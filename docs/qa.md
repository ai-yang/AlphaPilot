## 1. 核心引擎：LightGBM 模型的作用与预测机制

在本项目（基于 Qlib 的模板）中，**LightGBM（LGBModel）** 扮演着核心的“打分预测引擎”角色。

### 1.1 模型在框架中的具体职责
* **特征输入 (Feature)：** 接收经过公式计算后的量价或财务因子（如换手率、涨跌幅、动量等）。
* **目标拟合 (Label)：** 目标是预测股票未来的收益率（例如配置文件中的 `Ref($close, -2)/Ref($close, -1) - 1`，即 T+2 日开仓预期收益率）。
* **输出信号 (Prediction)：** 针对全市场股票，输出一个名为 `<PRED>` 的预期收益数值。分数越高，代表模型认为该股票未来上涨潜力越大。

### 1.2 数据是如何喂给模型的？（降维展平）
树模型无法直接处理“时间 × 股票 × 因子”的三维面板数据，因此需要进行**暴力展平**：
* **训练阶段（多天混合）：** 框架会将过去几年（如2019-2023）每天几千只股票的数据拼接到一张无限长的二维表格中。**模型在训练时是被抹去“日期”和“股票代码”概念的**，它只看“因子状态 -> 收益率”之间的非线性规律。
* **预测阶段（单日截面）：** 每天收盘后，系统将当天的全市场股票因子打包喂给模型。模型根据当下这**一瞬间**的因子状态，吐出明天的预测分数。
* **历史记忆的来源：** LightGBM 自身没有时间记忆。股票的“历史走势”（如连续上涨、放量突破）必须被预先写进**因子公式**里（例如计算过去 20 天的均值），浓缩成当天的数值喂给模型。

### 1.3 替代与进阶模型
* **树模型平替：** XGBoost、CatBoost、Random Forest（随机森林）。
* **进阶深度学习：** RNN / LSTM / Transformer（支持输入多天时间窗口的数据，无需完全展平）。
* **图神经网络（GNN）：** 捕捉上下游供应链或同行业股票之间的横向联动效应。

## 2. 数据降维与清洗：截面 Z-Score 标准化

因为模型把不同时间、不同走势的股票混在一起训练，必须抹平宏观噪音，这正是 **截面 Z-Score 标准化（Cross-Sectional Z-Score Norm）** 的核心使命。

### 2.1 什么是截面 Z-Score？
* **截面 (Cross-Sectional)：** 取同一天全市场所有股票的数据。
* **公式：** `Z-Score = (个股因子值 - 全市场平均值) / 全市场标准差`
* **含义：** 绝对的物理指标（如换手率 5%）被转化为了**相对全市场的排位分**（如比平均水平高 2 个标准差）。

### 2.2 为什么必须做截面处理？
1. **消除宏观大盘影响（Beta）：** 如果大盘暴涨，所有股票动量变大。减去均值后，水涨船高，Z-Score 不变。模型从而能精准识别出涨得比大盘还猛的“超额收益（Alpha）”股票。
2. **防止数据漂移：** 牛市市盈率 100 倍，熊市 15 倍。Z-Score 让跨越十年的数据处于同一基准线，“排名前 10%”的逻辑永远成立。
3. **方便机器学习收敛：** 将所有量纲不同的因子统一压缩到约 `-3` 到 `+3` 的范围内。

### 2.3 进阶处理细节
* **非连续型因子不做处理：** 行业分类、是否涨停等布尔值/离散类别特征不能做 Z-Score，否则会破坏语义。
* **鲁棒处理 (Robust Z-Score)：** 为了防止某只股票突发极端数据拉偏全市场均值，实际项目中常使用“中位数”代替均值，并配合截断（Clip Outliers）进行去极值处理。

## 3. 标准量化流水线：从原始数据到实盘选股

现代机器学习量化模型并不是单一的代码，而是一条严密的五步流水线：

1. **原始数据层 (Raw Data)：** 获取交易所提供的开高低收（OHLCV）及财务报表的三维面板数据。
2. **因子计算层 (Factor Construction)：** 利用数学公式（或大模型挖掘的公式），将原始价格数据转化为具有统计学意义的特征指标。
3. **数据清洗与标准化 (Preprocessing)：** 处理缺失值（停牌补齐），并进行去极值和**截面 Z-Score 标准化**，统一量纲。
4. **模型预测层 (Model Inference)：** 将清洗后的因子喂给 LightGBM 或神经网络。模型基于横向排序法则，给全市场股票打出 `<PRED>` 预期收益分。
5. **策略与回测执行 (Strategy & Backtest)：**
   * **选股策略：** 如 `TopkDropoutStrategy`，每天将 `<PRED>` 分数从高到低排序，无视股票名称，直接买入排名前 N（如 Top 100）的股票。
   * **回测验证：** 模拟真实账户买卖，扣除手续费（如千分之二）和滑点，计算夏普比率、最大回撤等绩效指标。

## 4. 终极进化：AlphaAgent 智能体自动化循环

传统的上述流水线中，“挖掘新因子”（第2步）往往需要人类研究员耗费数月时间。**AlphaAgent** 的核心理念是引入大语言模型（LLM），将“写代码->回测->改Bug”变成 7×24 小时的全自动闭环。

其 Agentic Workflow 包含以下五个无尽循环：

1. **构思提案 (Proposal)：** LLM 扮演研究员，结合金融先验知识，提出一种新的交易逻辑（如价量背离）。
2. **代码编写 (Coding)：** LLM 将自然语言想法翻译为 Qlib 框架可运行的代码表达式（如 `($close-Mean($close,20))/$volume`），并自动注入到 `conf.yaml` 模板中。
3. **回测执行 (Execution)：** 底层触发 Qlib 引擎，自动运行上述“计算因子 -> Z-Score -> LightGBM 打分 -> 模拟交易”的全套流水线。
4. **反思优化 (Feedback & Reflection)：** 系统将跑出的夏普比率、IC 值或报错日志反馈给大模型。大模型会自动修复 Bug，或者反思策略为何失效（例如：添加市值中性化处理）。
5. **知识沉淀 (Knowledge Management)：** 成功的高分因子被存入精华库；失败的逻辑被标记排雷，让 AI 随着时间推移越来越聪明。
